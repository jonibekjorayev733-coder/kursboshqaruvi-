# 🎬 QUICK REFERENCE - How to Use Each Feature\n\n## Admin Features\n\n### 1️⃣ Manage Students (AdminStudents)\n```\n📍 Path: /admin/students\n🎨 Design: Blue-Cyan-Teal gradient\n\n✅ FEATURES:\n• View all students in card grid\n• Search students by name or email\n• EDIT student: Click Edit button → Modal form → Update fields\n• DELETE student: Click Delete button → Confirm → Student removed\n• BULK DELETE: Select multiple students → \"O'chirish\" button → Confirm\n\n💡 KEYBOARD SHORTCUTS:\n- Ctrl+F → Focus search\n- Esc → Close modal\n```\n\n### 2️⃣ Manage Teachers (AdminTeachers)\n```\n📍 Path: /admin/teachers\n🎨 Design: Purple-Pink-Red gradient\n\n✅ FEATURES:\n• View all teachers in card grid\n• Assign courses to teachers\n• EDIT teacher: Click Edit button → Modal form → Update info\n• DELETE teacher: Click Delete button → Confirm → Teacher removed\n• See assigned courses count\n\n💡 TIP: When editing, change course assignment if needed\n```\n\n### 3️⃣ Manage Courses (AdminCourses)\n```\n📍 Path: /admin/courses\n🎨 Design: Green-Emerald-Teal gradient\n\n✅ FEATURES:\n• View all courses with student count\n• See course progress (if applicable)\n• EDIT course: Click Edit button → Modal form → Update details\n• DELETE course: Click Delete button → Confirm → Course removed\n• View assigned teacher\n\n💡 TIP: Be careful deleting courses with students enrolled\n```\n\n### 4️⃣ Manage Payments & Send SMS (AdminPayments) ⭐ KEY FEATURE\n```\n📍 Path: /admin/payments\n🎨 Design: Amber-Orange-Red gradient\n\n✅ FEATURES:\n• View all student payments\n• Filter by status (All, Paid, Pending, Failed)\n• See 4 metrics: Total Paid, Pending, Failed, Paid Count\n\n📌 CLICK ON PAYMENT ROW:\n→ Sidebar appears from right\n→ Shows student details:\n   • Avatar + name + ID\n   • Email, Phone, Telegram (clickable links)\n   • Payment details (course, month, amount, status)\n\n🎯 SEND SMS TO STUDENT:\n1. Click on a pending payment row\n2. Sidebar appears with student info\n3. Click \"SEND SMS\" button\n4. Loading animation starts (1 second)\n5. SMS sent notification appears\n6. Student receives notification instantly\n7. Payment status simulates update\n\n📊 NOTIFICATIONS PANEL:\n• Shows green panel below header\n• Displays when students make payments\n• Badge shows unread count (red)\n• Click \"Ko'rildi\" to mark as read\n\n💡 TIP: Watch StudentPayments window to see notifications sync in real-time!\n```\n\n---\n\n## Student Features\n\n### 1️⃣ View & Make Payments (StudentPayments) ⭐ KEY FEATURE\n```\n📍 Path: /student/payments (in student role)\n🎨 Design: Cyan-Blue-Indigo gradient\n\n✅ FEATURES:\n• View 3 key metrics:\n  - Total Paid (green)\n  - Total Due (amber)\n  - Pending Count (blue)\n\n• See all payments in list:\n  - Course name\n  - Month\n  - Amount\n  - Status badge (Paid/Pending)\n  - Due date warning (red, if pending)\n\n📌 RECEIVE ADMIN SMS REMINDERS:\n• Blue notification panel shows SMS from admin\n• Bell icon with badge showing unread count\n• Click \"Mark as Read\" to acknowledge\n\n🎯 MAKE A PAYMENT:\n1. Open two browser windows/tabs\n2. Window 1: Go to /admin/payments (as admin)\n3. Window 2: Go to /student/payments (as student)\n\n**Step 1: Admin views payments (1 min)**\n```\nWindow 1:\n├─ See payment table with students\n├─ Click a payment row\n├─ Sidebar appears with student info\n├─ Notice \"SEND SMS\" button\n└─ Note the student name and email\n```\n\n**Step 2: Admin sends SMS (2 min)**\n```\nWindow 1:\n├─ Click \"SEND SMS\" button\n├─ Watch loading animation\n├─ See green notification appear\n└─ Check red badge on header\n\nWindow 2 (WATCH THIS!):\n├─ Blue notification panel updates\n├─ SMS reminder appears\n├─ Bell icon shows badge\n└─ Student now knows to pay\n```\n\n**Step 3: Student makes payment (2 min)**\n```\nWindow 2:\n├─ Find the pending payment\n├─ Click \"To'lovni Qil\" button\n├─ Watch 2-second loading\n├─ Status changes to \"Tulangan\"\n└─ See toast confirmation\n\nWindow 1 (WATCH THIS!):\n├─ Green notification updates\n├─ Shows \"Student made payment\"\n├─ Red badge increases\n├─ Payment table reflects change\n└─ Status is now \"paid\"\n```\n\n**Step 4: Mark as read (1 min)**\n```\nWindow 1:\n├─ In green notification panel\n├─ Click \"Ko'rildi\" button\n├─ Notification becomes gray\n└─ Badge count decreases\n\nWindow 2:\n├─ In blue notification panel\n├─ Click \"Mark as Read\"\n├─ Notification becomes gray\n└─ Badge count decreases\n```\n\n---\n\n## 🛠️ Troubleshooting\n\n### Issue: Notifications not appearing?\n```\n✅ SOLUTION:\n1. Check if AppContext is properly wrapped\n2. Verify both windows are on same domain\n3. Check browser console for errors\n4. Try refreshing the page\n```\n\n### Issue: Delete/Edit buttons not working?\n```\n✅ SOLUTION:\n1. Check if modal appears (might be off-screen)\n2. Verify all required fields in form\n3. Check browser console for API errors\n4. Try with different student/teacher/course\n```\n\n### Issue: SMS not triggering payment update?\n```\n✅ SOLUTION:\n1. Wait 1 second for SMS simulation\n2. Payment updates automatically in 2 seconds\n3. Refresh page if needed\n4. Check browser network tab for API calls\n```\n\n### Issue: Real-time sync not working?\n```\n✅ SOLUTION:\n1. Open windows in same browser (not private mode)\n2. Check AppContext is loaded\n3. Console should show no errors\n4. Try closing and reopening both windows\n```\n\n---\n\n## 📱 Mobile Testing\n\n### Features that work on mobile:\n✅ All payment features\n✅ All delete/edit modals\n✅ Notifications\n✅ SMS reminders\n✅ Responsive design (1 column on mobile)\n\n### To test on mobile:\n1.
